# Smart Classroom Timetable Scheduler Using Machine Learning

## Project Report

---

## 1. Abstract

This project presents an intelligent timetable scheduling system that leverages machine learning and constraint-based optimization to automatically generate conflict-free academic schedules. The system combines Constraint Satisfaction Problems (CSP), Genetic Algorithms (GA), Decision Trees, and K-Means Clustering into a unified ML pipeline that handles classroom allocation, faculty scheduling, subject prioritization, and time slot optimization. The system achieved zero-conflict timetable generation for 70+ classes across 6 semesters with a fitness score of 2367.3, completing in under 1 second.

---

## 2. Introduction

### 2.1 Background

Timetable scheduling is a critical operational challenge in educational institutions. It involves allocating limited resources—classrooms, faculty, and time slots—to meet the demands of multiple courses, semesters, and departments while satisfying numerous hard and soft constraints. This problem is classified as NP-hard in computational complexity theory, making optimal solutions computationally expensive to find.

### 2.2 Motivation

Traditional timetable generation relies on manual effort or rigid rule-based systems that:
- Produce frequent scheduling conflicts
- Fail to optimize resource utilization
- Cannot adapt to dynamic changes
- Scale poorly with institutional growth

Machine learning offers a data-driven approach that can learn from historical patterns, adapt to changing constraints, and continuously improve scheduling quality.

### 2.3 Scope

This project delivers a complete web-based application with:
- Automated timetable generation using a hybrid ML pipeline
- Admin, Faculty, and Student portals
- CRUD operations for all academic entities
- Real-time analytics and reporting
- Swap request management
- Historical data learning

## 3. Literature Survey

### 3.1 Classical Approaches

| Approach | Strengths | Limitations |
|----------|-----------|-------------|
| Manual Scheduling | Human intuition, flexibility | Time-consuming, error-prone, doesn't scale |
| Graph Coloring | Theoretically sound | Oversimplifies real constraints |
| Integer Linear Programming | Optimal for small instances | Computationally expensive for large inputs |

### 3.2 Metaheuristic Approaches

| Algorithm | Application | Effectiveness |
|-----------|-------------|---------------|
| Genetic Algorithm | Multi-objective optimization | Good balance of exploration/exploitation |
| Simulated Annealing | Local search optimization | Can escape local optima |
| Ant Colony Optimization | Path-based scheduling | Effective for routing-style problems |
| Particle Swarm Optimization | Continuous optimization | Less suited for discrete scheduling |

### 3.3 ML-Based Approaches

Recent research explores:
- **Reinforcement Learning** for adaptive scheduling (reward-based learning)
- **Neural Networks** for pattern recognition in historical data
- **Decision Trees** for interpretable rule extraction
- **Clustering** for resource grouping and utilization analysis

### 3.4 Our Approach

We adopt a **hybrid pipeline** combining CSP (for feasibility), GA (for optimization), Decision Trees (for pattern learning), and Clustering (for resource analysis). This gives us the benefits of both constraint-based correctness and ML-driven optimization.

## 5. Implementation

### 5.1 ML Pipeline - Algorithm Details

#### 5.1.1 Constraint Satisfaction Problem (CSP) Solver

**Purpose:** Generate a valid, conflict-free timetable as the baseline.

**Why CSP:** Timetable scheduling is fundamentally a constraint satisfaction problem. Hard constraints (no teacher in two places at once, no room double-booking) must be absolutely satisfied. CSP guarantees this.

**Algorithm:**
1. Define variables: Each (subject, lecture_index) pair
2. Define domains: All possible (day, timeslot, room, faculty) tuples
3. Apply constraints to prune domains
4. Use greedy assignment with random restarts
5. Variables with fewer valid options (MRV heuristic) are assigned first

**Hard Constraints Enforced:**
- No faculty teaching two classes simultaneously
- No classroom double-booking
- No semester with two classes at same time
- Room capacity must meet student count
- Labs must be in lab rooms
- Faculty daily/weekly hour limits

```
For each variable (subject, lecture_index):
    Sort by priority (MRV heuristic)
    For each value in domain (day, slot, room, faculty):
        If all_constraints_satisfied:
            Assign value
            Break
    If unassigned:
        Restart with random shuffling
```

#### 5.1.2 Genetic Algorithm (GA)

**Purpose:** Optimize the CSP-generated timetable for quality.

**Why GA:** CSP produces a *valid* schedule but not necessarily an *optimal* one. GA evolves the solution toward multiple soft objectives simultaneously.

**Fitness Function (Multi-Objective):**
```
fitness = 1000
    - conflict_penalty * 100    (hard: should be 0)
    + preference_score * 2       (faculty teaching preferred subjects)
    + workload_balance * 3       (even distribution across days)
    - gap_penalty * 1.5          (minimize student free periods)
    + distribution_score * 2     (spread subjects across week)
```

**Operators:**
- **Selection:** Tournament selection (size 3)
- **Crossover:** Semester-based single-point crossover
- **Mutation:** Random day/timeslot/room swaps (15% rate)
- **Elitism:** Top 3 individuals preserved each generation

**Parameters:**
- Population: 20-25 individuals
- Generations: 50-100
- Crossover rate: 80%
- Mutation rate: 15%

#### 5.1.3 Decision Tree (Pattern Learning)

**Purpose:** Learn which time-slot + room + faculty combinations historically produced the best outcomes.

**Why Decision Tree:**
- Interpretable: admins can understand WHY certain slots are preferred
- Fast inference for real-time suggestions
- Works well with small-to-medium historical datasets

**Features (8 input features):**
| Feature | Type | Description |
|---------|------|-------------|
| subject_id | Categorical | Subject identifier |
| faculty_id | Categorical | Faculty identifier |
| day_of_week | Ordinal (0-4) | Day of scheduling |
| timeslot_id | Ordinal | Time period |
| is_lab | Binary | Whether subject is lab |
| priority | Ordinal (1-10) | Subject priority |
| student_count | Numeric | Students in semester |
| room_capacity | Numeric | Room size |

**Target:** fitness_score (regression) and good_slot (classification)

**Results from testing:**
- Regression MSE: 0.0037
- Classification Accuracy: 94%
- Most important features: timeslot_id (32.8%), room_capacity (30.7%)

#### 5.1.4 K-Means Clustering (Room Analysis)

**Purpose:** Group classrooms by usage patterns to identify optimization opportunities.

**Why Clustering:**
- Identifies underutilized rooms → prioritize them in scheduling
- Groups rooms by usage profile → inform allocation strategy
- Detects peak-hour bias → help with load balancing

**Features per room:**
1. Occupancy rate (used slots / total slots)
2. Average capacity ratio (class size / room capacity)
3. Subject variety (unique subjects / total classes)
4. Day spread (days used / 5)
5. Peak hour usage (morning classes / total classes)

**Clusters:** 3 groups (low_usage, medium_usage, high_usage)

## 7. Conclusion

The Smart Classroom Timetable Scheduler successfully demonstrates that a hybrid ML approach combining CSP, Genetic Algorithms, Decision Trees, and Clustering can:

1. **Generate conflict-free timetables** in under 1 second for a realistic institutional dataset
2. **Optimize multiple objectives** simultaneously (preferences, workload balance, gap minimization)
3. **Learn from historical data** to improve future scheduling decisions
4. **Provide actionable insights** through feature importance analysis and utilization clustering
5. **Scale effectively** to handle multiple departments, courses, and semesters

The zero-conflict result validates the CSP solver's correctness, while the GA's convergence from 2291 to 2367 fitness demonstrates meaningful optimization beyond mere feasibility.

4. **Student Preference Integration:** Allow students to express elective preferences, with the scheduler optimizing group formation.
